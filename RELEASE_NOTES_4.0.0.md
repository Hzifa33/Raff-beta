# ملاحظات إصدار رَفّ 4.0.0

رَفّ 4 هو انتقال من مدير كتب محلي إلى نواة نظام مكتبات عربي متعدد المساحات والأدوار، مع الحفاظ على التشغيل دون إنترنت والتوافق مع بيانات 2.7.

## أهم ما يقدمه الإصدار

- سجل ببليوغرافي، مقتنيات، ونسخ مادية مستقلة.
- مستعيرون وسياسات وإعارة وتجديد وحجوزات وإرجاع جزئي.
- جرد بالباركود ونقل بين الفروع.
- فهرس باحث وOPAC محلي.
- ضبط استنادي وصيغ تبادل مكتبية.
- تزويد ودوريات ووضع دار نشر.
- مستخدمون وصلاحيات وسجل تدقيق وسلة محذوفات.
- نسخ احتياطية وفحص سلامة وإصلاح آمن.
- واجهة منظمة حسب دور الباحث أو موظف الإعارة أو المفهرس أو المدير.

## التحقق المنفذ

- فحص صياغة جميع ملفات JavaScript الرئيسية.
- اختبارات سلامة الإصدار السابق.
- اختبارات نطاق رَفّ 4.
- اختبارات عقد الواجهة والعمل المحلي.
- تدقيق اعتماديات الإنتاج: 0 ثغرات معلنة في نتيجة `npm audit --omit=dev` أثناء إعداد الحزمة.

## بناء Windows

ملف المصدر جاهز للبناء عبر GitHub Actions أو جهاز متصل بالإنترنت. لم يُنتج ملف EXE داخل بيئة إعداد الحزمة لأن electron-builder لم يتمكن من الوصول إلى GitHub لتنزيل ثنائية Electron، وفشل بطلب DNS `EAI_AGAIN`. هذا فشل بيئي في التنزيل، وليس فشلًا في الاختبارات أو صياغة المصدر.

## الاستثناءات المقصودة

- لا ذكاء اصطناعي.
- لا جلب ISBN أو أغلفة من الإنترنت.
- لا مزامنة سحابية.
- لا تحليلات استخدام أو تتبع.
- RFID وSIP2 وNCIP ليست تكاملات معتمدة ميدانيًا دون أجهزة وأنظمة طرف ثالث.
