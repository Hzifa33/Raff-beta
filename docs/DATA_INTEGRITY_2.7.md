# تدقيق سلامة البيانات في رَفّ 2.7.0

## قاعدة مقارنة الرقم المرجعي

تُبنى هوية الرقم المرجعي من **القيمة الكاملة** بعد إزالة فروق التنسيق غير المقصودة فقط:

- المسافات في البداية والنهاية.
- المحارف غير المرئية واتجاهات النص المضمّنة.
- أشكال الشرطة المختلفة، مثل `–` و`—`، وتحويلها إلى `-`.
- المسافات حول الشرطة.
- الأرقام العربية والفارسية وتحويلها إلى أرقام لاتينية موحّدة.
- حالة الأحرف اللاتينية عند المقارنة.

لا يُستخدم التطابق الجزئي ولا آخر أرقام السلسلة وحدها. لذلك:

- `raf-0001` و`raf-1001`: **مختلفان**.
- `RAF – 0001` و`raf-1` و`raf-0001`: **هوية واحدة** مع اختلاف تنسيق.
- `RAF-TEST-1001` لا يؤثر في تسلسل أرقام `raf-0001` التلقائي.

يُستخدم هذا المنطق نفسه في:

1. إضافة كتاب.
2. تعديل الرقم المرجعي.
3. استيراد JSON.
4. فحص سلامة البيانات.
5. البحث اليدوي وقارئ الباركود.

## الفحص البنيوي

يفحص النظام:

- الأرقام المرجعية المكررة فعليًا.
- الأرقام الناقصة أو التي تحتاج توحيد تنسيق.
- معرّفات الكتب المكررة أو المفقودة.
- الحقول الأساسية الناقصة.
- عدد النسخ والأجزاء.
- بنية سجل الإعارات ومعرّفات الإعارات.
- تواريخ الإعارة والاستحقاق والإرجاع.
- الأجزاء المسجلة في الإعارات مقارنة بعدد أجزاء الكتاب.
- عدد الإعارات المفتوحة مقارنة بعدد النسخ.

أما الإعارات المتأخرة والعناوين المتشابهة فتظهر كتنبيهات للمراجعة، ولا تُعامل بوصفها تلفًا بنيويًا في قاعدة البيانات.

## الإصلاح الآمن

قبل أي إصلاح، يُنشئ البرنامج نسخة JSON كاملة داخل مجلد النسخ الاحتياطية. بعد ذلك يمكنه تلقائيًا:

- توحيد التنسيق الآمن للأرقام المرجعية.
- إبقاء أقدم سجل عند تكرار الرقم، ومنح السجلات الأخرى أقل أرقام `raf` المتاحة.
- منح رقم مرجعي للسجل الذي لا يملك رقمًا.
- إصلاح معرّفات الكتب والإعارات المكررة أو المفقودة.
- تصحيح القيم الرقمية الحتمية للنسخ والأجزاء.
- إنشاء تاريخ استحقاق صحيح إذا كان مفقودًا أو تالفًا.

لا يحذف الإصلاح الآمن أي كتاب أو إعارة، ولا يحاول تخمين العنوان أو المؤلف أو المجال، ولا يغلق إعارة ذات تاريخ إرجاع غير صالح تلقائيًا.
