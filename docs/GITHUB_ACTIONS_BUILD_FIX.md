# إصلاح بناء GitHub Actions

كان ملف `package-lock.json` يحتوي على روابط `resolved` خاصة بسجل حزم داخلي لبيئة البناء، لذلك لم يستطع GitHub Actions تنزيل الاعتماديات بصورة صحيحة وانتهى `npm ci` برسالة عامة: `Exit handler never called!`.

تم الإصلاح عبر:

- تحويل جميع روابط الحزم إلى `https://registry.npmjs.org/`.
- إضافة `.npmrc` يثبت السجل العام.
- تحديث إجراءات GitHub إلى إصدارات تعمل على Node.js 24 داخليًا.
- تثبيت Node.js 22.23.1 لتشغيل أوامر المشروع.
- فحص ملف القفل ومنع أي رابط داخلي قبل التثبيت.
- إضافة إعادة محاولة محدودة عند فشل الشبكة المؤقت.
- إبقاء `npm ci` مع ملف القفل لضمان بناء قابل للتكرار.
## إصلاح غياب ملفات الاختبارات

فشل `npm test` لاحقًا لأن مجلد `tests/` لم يكن موجودًا داخل المستودع على GitHub، رغم أن `package.json` يستدعي ثلاثة ملفات منه. أضيفت الملفات التالية إلى حزمة الإصلاح:

- `tests/integrity.test.js`
- `tests/v4.test.js`
- `tests/ui-contract.test.js`

كما أضيفت خطوة مبكرة إلى سير العمل تتحقق من وجود ملفات المصدر والاختبارات المطلوبة، وتعرض قائمة الملفات المتتبعة عند غياب أي ملف بدل الوصول إلى خطأ `MODULE_NOT_FOUND` غير الواضح.
