# ترقية بيانات رَفّ 2.7 إلى 4.0

## قبل الترقية

1. أغلق رَفّ 2.7.
2. احتفظ بنسخة من مجلد بيانات البرنامج.
3. احتفظ بآخر تصدير JSON معروف السلامة.

## ما يحدث تلقائيًا

عند فتح 4.0 لأول مرة:

- يُنشأ نموذج `raff4` إن لم يكن موجودًا.
- يتحول كل كتاب قديم إلى سجل ببليوغرافي.
- يُنشأ مقتنى افتراضي في الفرع الرئيسي.
- تُنشأ نسخة مادية لكل تركيب من عدد النسخ وعدد الأجزاء.
- تتحول الإعارات القديمة إلى قروض مرتبطة بالمستعيرين والنسخ.
- تُحفظ معرفات الربط مع بيانات 2.7.
- تُرقى البنية دون حذف الملف القديم.

## بعد الترقية

- شغّل فحص سلامة البيانات.
- راجع السجلات متعددة الأجزاء.
- راجع الباركودات المتولدة.
- أنشئ نسخة احتياطية باسم واضح.
- اختبر إعارة وإرجاع نسخة تجريبية.

## الرجوع

لا تستخدم ملف 4.0 المعدل داخل إصدار أقدم دون نسخة منفصلة. الرجوع الآمن يكون باستعادة نسخة مجلد البيانات التي أُنشئت قبل الترقية.
