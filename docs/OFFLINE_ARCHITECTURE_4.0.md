# المعمارية المحلية في رَفّ 4.0

## المبدأ

كل وظيفة أساسية تعمل من دون إنترنت. لا توجد طلبات إلى API خارجية، ولا تسجيل دخول سحابي، ولا تتبع.

## الطبقات

```text
Renderer RTL UI
    ↓ preload API محدود
Electron Main Process
    ↓
RaffV4Store + Legacy Store Compatibility
    ↓
ملف بيانات محلي ونسخ احتياطية
```

## OPAC المحلي

يبدأ خادم HTTP محلي عند طلب المدير فقط. الوضع الافتراضي `localhost`، ويمكن السماح للشبكة المحلية صراحة. يعرض بيانات الفهرس العامة فقط ولا يعرض المستعيرين أو الإعارات أو المستخدمين.

## التبادل بين الأجهزة

بدل مزامنة سحابية، تصدر حزمة `raff-offline-transfer` يمكن نقلها بوسيط محلي ثم استيرادها. الدمج الحالي يضيف الكيانات الجديدة حسب المعرف؛ يجب أخذ نسخة احتياطية قبل الدمج.

## الأمان

- Renderer معزول ولا يملك Node.js مباشرًا.
- Electron sandbox مفعّل.
- IPC محدد بالوظائف اللازمة.
- النوافذ والتنقل والروابط الخارجية مقيدة.
- PIN مشتق باستخدام `scrypt`.
- CSP يمنع تحميل خطوط وسكربتات بعيدة.

## التخزين

يستخدم الإصدار الحالي مخزن JSON محليًا لضمان سهولة النقل وعدم الحاجة إلى اعتماديات أصلية. هذا مناسب للتشغيل الفردي والمكتبات الصغيرة والمتوسطة. الانتقال إلى SQLite متعدد المعاملات موثق كمسار مستقبلي، لكنه غير مفروض في 4.0 حتى لا تُكسر قابلية النقل والبناء.
