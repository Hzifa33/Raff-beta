# تحليل تطبيق لغة shadcn/ui على «رَفّ»

## القرار المعماري

المشروع الحالي تطبيق Electron مكتوب بـ HTML وCSS وJavaScript دون React. لذلك لم تُحوَّل البنية إلى React ولم تُستبدل طبقة التخزين أو المسارات أو نماذج البيانات. التحويل إلى React لمجرد استخدام مكوّنات shadcn كان سيضيف مخاطرة غير ضرورية على الإعارات، البحث، الاستيراد، التصدير، الباركود، النسخ الاحتياطي، وسجل الأجزاء.

بدلاً من ذلك جرى تحليل مستودع `shadcn-ui/ui` المرفق، ثم نقل **نظام التصميم نفسه** إلى طبقة CSS أخيرة مستقلة باسم `src/css/shadcn-raff.css`.

## ما تم تحليله من المستودع

- بنية المتغيرات الدلالية في `globals.css`: `background`, `foreground`, `card`, `popover`, `primary`, `secondary`, `muted`, `accent`, `destructive`, `border`, `input`, `ring`, ومتغيرات `sidebar`.
- عقود المكوّنات الأساسية: Button، Card، Input، Table، Dialog، Badge وSidebar.
- قواعد الحالات: hover، focus-visible، disabled، destructive، selected وdark mode.
- مبدأ الحواف الهادئة ذات البكسل الواحد، وحلقة التركيز ثلاثية البكسل، وتقليل الظلال والتدرجات.
- كثافة المقاسات: أزرار وحقول مدمجة، بطاقات بمسافات داخلية ثابتة، وجداول تعتمد الصفوف لا «بطاقات داخل بطاقات».

## خريطة التطبيق

| shadcn/ui | رَفّ |
|---|---|
| `background` | خلفية مساحة العمل |
| `card` | اللوحات، بطاقات الإحصاء، أقسام الإعدادات |
| `popover` | النوافذ المنبثقة، الإكمال التلقائي، التنبيهات |
| `primary` | الهوية الذهبية والأزرار الأساسية |
| `muted` | الصفوف عند المرور، خلفيات التقسيم، عناصر ثانوية |
| `accent` | الرقم المرجعي والعنصر المحدد |
| `destructive` | الحذف والتأخر والأخطاء |
| `border` | كل الحدود الهادئة الموحدة |
| `input` | حدود حقول الإدخال |
| `ring` | حلقة التركيز وإمكانية الوصول |
| `sidebar-*` | القائمة الجانبية البنية وحالاتها |

## المحافظة على هوية «رَفّ»

الثيم ليس نسخة رمادية عامة. تم تحويل لوحتي shadcn النهارية والليلية إلى درجات دافئة:

- النهاري: رقّ هادئ، بطاقات عاجية، نص بني، ذهب نحاسي.
- الليلي: أسود بني غير فاحم، بطاقات جوزية داكنة، نص كريمي، ذهب أكثر إشراقاً.
- القائمة الجانبية تحتفظ بالهوية البنية في الوضعين، بينما تتغير بقية مساحة العمل بالكامل بين النهاري والليلي.

## ما لم يتغير

- ملفات التخزين والفهرسة والبحث.
- بنية الكتاب والنسخ والإعارات.
- إعارة مجموعة كاملة أو جزء محدد من كتاب متعدد الأجزاء.
- الاستدعاء والإحصائيات والتصدير والطباعة والباركود.
- ترتيب الصفحات والقائمة القابلة للطي والاختصارات.
- جميع معرفات DOM ومستمعات الأحداث؛ طبقة الثيم بصرية فقط.

## قواعد الصيانة

1. أضف أي ألوان جديدة كمتغيرات دلالية، لا كقيم مباشرة داخل المكوّن.
2. استخدم `card` للمحتوى، و`popover` للعناصر العائمة فقط.
3. استخدم `muted` للمرور والعناصر الثانوية، و`accent` للتحديد.
4. لا تضف تدرجات أو Glassmorphism إلا لسبب وظيفي واضح.
5. يجب أن يحمل أي عنصر قابل للتفاعل حالة `focus-visible` واضحة.
