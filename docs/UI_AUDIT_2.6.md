# تدقيق واجهة رَفّ 2.6.0

## نطاق المراجعة

شملت المراجعة الشريط الجانبي، لوحة المعلومات، البحث المتقدم، الإضافة، السجل الكامل، المسح، الاستدعاء، الإحصائيات، الإعدادات، والوضعين النهاري والليلي.

## المشكلات التي عولجت

- فصل شعار الشريط وزر الطي وأول عنصر تنقّل إلى مساحات مستقلة عند الطي.
- نقل تلميحات أسماء الأقسام إلى عنصر عائم خارج الحاويات ذات `overflow: hidden` لمنع قصها.
- استبدال رمز السجل الكامل برمز جدولي متماثل بصريًا.
- إعادة تسمية القسم إلى «الإعدادات والنسخ».
- إضافة ترتيب مباشر حسب وقت الإضافة صعودًا وهبوطًا وإظهار الساعة داخل الجدول.
- جعل أعمدة السجل تعتمد على العرض الفعلي لمساحة المحتوى، مع مستويات أولوية تمنع التمرير الأفقي.
- إزالة تغيير العرض اليدوي للأعمدة لأنه كان يسمح بإنشاء تجاوز أفقي غير متوقع.
- إضافة حالة `aria-current` للقسم النشط وتحسين التركيز بلوحة المفاتيح وتقليل الحركة عند طلب النظام.

## اختبارات الواجهة

- 1600×900: الشريط مطوي، لا تداخل بين الشعار وزر الطي وأول أيقونة، والتلميح كامل داخل الشاشة.
- 1366×768: السجل الكامل موسّع، عناوين الأعمدة غير مقصوصة ولا يوجد تمرير أفقي، وترتيب «الأحدث إضافة» يعرض أحدث سجل أولًا.
- 1100×700: فحص جميع الصفحات الرئيسية دون تجاوز أفقي في `body` أو العمود الرئيسي أو مساحة العرض.
- الوضع الليلي: السجل الكامل والشريط المطوي دون تجاوز أفقي أو خلل في التباين.
- فحص صياغة جميع ملفات JavaScript باستخدام `node --check`.

## التوافق

لم يتغير مخطط قاعدة البيانات أو منطق التخزين والإعارة والعودة الجزئية والبحث والتقارير والباركود، ولذلك تظل بيانات الإصدارات السابقة متوافقة.
