# إصلاح ثبات الواجهة والقوائم — رَفّ 4.0.1

## المشكلة

كانت حاوية العرض الرئيسية تعمل كعمود Flex بارتفاع النافذة، بينما كانت صفحات رَفّ 4 نفسها عناصر قابلة للانكماش. عند استخدام شاشة قصيرة أو تكبير Windows بنسبة 125% أو 150% كانت الأقسام الكبيرة تضغط رأسيًا حتى تصبح بطاقات الترحيب ومسار العمل وبعض اللوحات مجرد شريط رفيع يخفي محتواها.

## الإصلاح

- أصبحت صفحة المسار تنمو حسب ارتفاع محتواها الحقيقي.
- مُنع انكماش أقسام الصفحة والبطاقات والشبكات داخل مساحة التمرير.
- بقي التمرير داخل `view-scroll` بدل ضغط الأقسام لتناسب النافذة.
- استُثنيت الشاشات التي تمتلك تمريرًا داخليًا مقصودًا، مثل السجل الكامل والتقارير، للحفاظ على أدائها.
- أضيفت قواعد خاصة للشاشات القصيرة ودرجات تكبير Windows.

## القوائم المنسدلة

استُبدلت قوائم `<select>` المرئية بمكوّن محلي مخصص لواجهة رَفّ:

- التصميم متوافق مع الهوية البنية والذهبية والوضعين النهاري والليلي.
- القائمة تُعرض فوق جميع البطاقات والنوافذ ولا تتعرض للقص بسبب `overflow`.
- القوائم الطويلة تحتوي بحثًا داخليًا تلقائيًا.
- دعم كامل للوحة المفاتيح: الأسهم، Home، End، Enter، Escape وTab.
- يبقى عنصر `<select>` الأصلي مصدر الحقيقة داخل النموذج؛ لذلك تظل `FormData` والتحقق والأحداث الحالية تعمل دون تغيير.
- أي قائمة تُنشأ ديناميكيًا داخل النوافذ أو الصفحات تُحوّل تلقائيًا بواسطة `MutationObserver`.
- لا توجد مكتبات أو اتصالات خارجية.

## شريط النافذة

أُعيد ترتيب عناصر شريط النافذة في RTL ليصبح زر الإغلاق في أقصى اليمين، يليه التكبير ثم التصغير.
