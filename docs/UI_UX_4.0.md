# دراسة UI/UX لرَفّ 4.0

## الهدف

المشكلة الأساسية في أنظمة المكتبات الكبيرة ليست نقص الوظائف، بل عرض كل الوظائف لكل مستخدم في الوقت نفسه. لذلك بُنيت واجهة رَفّ 4 على مبدأ **الإفصاح التدريجي**: يرى المستخدم ما يحتاجه الآن، وتبقى الأدوات المتقدمة في مساحة مستقلة.

## شخصيات الاستخدام

### الباحث أو الزائر

يحتاج إلى البحث، ومعرفة مكان الكتاب وإتاحته، وبناء قائمة قراءة. لا يحتاج إلى بيانات المستعيرين أو المشتريات أو سلامة قاعدة البيانات.

### موظف الإعارة

يحتاج إلى تحديد المستعير، ومسح النسخ، وتنفيذ الإعارة أو الإرجاع أو التجديد، ومتابعة الحجوزات. لا يحتاج إلى محرر MARC أو إعدادات النظام.

### المفهرس

يحتاج إلى السجل الببليوغرافي، والمقتنيات، والنسخ، والضبط الاستنادي، والاستيراد والتصدير. لا يحتاج إلى بيانات تواصل القراء.

### المدير وأمين المكتبة

يحتاجان إلى السياسات، والفروع، والمستخدمين، والتقارير، والتدقيق، والنسخ الاحتياطية وسلامة البيانات.

### مسؤول دار النشر

يحتاج إلى الإصدارات والطبعات والمخزون والتزويد والتقارير، دون ازدحام وظائف الإعارة اليومية.

## بنية التنقل

تقسم القائمة إلى مجموعات ثابتة:

1. الرئيسية والبحث.
2. خدمة القراء.
3. الفهرسة والمجموعات.
4. التزويد والنشر.
5. الإدارة.

يحدد الدور مساحة العمل الفعلية لغير المدير، وتُخفى الصفحات غير المصرح بها. لا يعتمد التنظيم على قائمة طويلة مسطحة.

## قواعد النماذج

- الحقول الأساسية أولًا.
- الحقول المتقدمة في أقسام منطقية.
- الحقول المخصصة تظهر في نطاقها فقط.
- لا يُفقد نموذج غير محفوظ بصمت.
- الأرقام مثل ISBN والباركود تعرض باتجاه LTR.
- رسائل الخطأ تصف الإجراء المطلوب بدل عبارات عامة.
- العمليات الخطرة تحتاج تأكيدًا ونسخة احتياطية عند ملاءمة ذلك.

## رحلة الكتاب

1. إنشاء سجل ببليوغرافي.
2. إنشاء مقتنى يحدد الفرع والموقع.
3. إنشاء نسخة أو أكثر، ولكل نسخة باركود مستقل.
4. إتاحة النسخة في الفهرس.
5. إعارة النسخة أو حجزها أو نقلها أو جردها.
6. ظهور كل تغيير في سجل التدقيق والتقارير.

## رحلة الباحث

1. فتح فهرس الباحث مباشرة.
2. إدخال كلمة واحدة أو استخدام المرشحات.
3. رؤية العنوان والمؤلف والإتاحة دون مصطلحات إدارية.
4. فتح التفاصيل لمعرفة الفرع والرف.
5. إضافة العنوان إلى قائمة قراءة.

## رحلة الإعارة

1. اختيار المستعير أو مسح بطاقته.
2. مسح نسخة واحدة أو عدة نسخ.
3. عرض القيود قبل التنفيذ.
4. تأكيد الإعارة وإظهار موعد الاستحقاق.
5. في الإرجاع، يكفي مسح الباركود دون فتح السجل.

## الوصول

- تنقل بلوحة المفاتيح وحالات تركيز واضحة.
- دعم تقليل الحركة.
- تباين مرتفع اختياري.
- تكبير الواجهة من 80% إلى 150% في طبقة البيانات، ومن 90% إلى 125% في واجهة الإعدادات الحالية.
- كثافة مريحة أو مضغوطة.
- عدم الاعتماد على اللون وحده؛ كل حالة لها نص.

## منع الفوضى

- الباحث لا يرى أدوات الإدارة.
- الإجراءات الثانوية داخل تفاصيل السجل، لا في كل صف.
- الإحصاءات تعرض الأهم أولًا.
- التنبيهات في مركز مستقل.
- إعدادات الوحدات منفصلة عن إعدادات المظهر.
- أدوات التبادل مجمعة في صفحة واحدة.
- الوضع التقليدي محفوظ للتوافق لكنه غير ظاهر في التنقل الرئيسي.
